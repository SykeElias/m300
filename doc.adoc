*SSH Parameter:*

    -i    Identity-File definieren (privater Schlüssel)
    -X    Graphische Oberfläche remote ansprechbar machen (X-Forwarding)
    -F    Konfigurationsfile angeben
    -o    einzelne Option anpassen
    -N    kein Remote-Kommando ausführen
    -T    kein Terminal verwenden
    -p    Verbindung auf Portnummer aufbauen (statt Port 22)
    -v    Debug-Ausgabe (auch mehrfach: -vvv)

*IP-Adressen Referenz:*

    0.0.0.0       alle Interfaces
    127.0.0.1     Localhost (nur lokal erreichbar)
    ::1           Localhost IPv6
    192.168.1.1   statische Adresse eines Hosts

== DNS

*Wichtigste DNS Record-Typen:*

[cols="1,3"]
|===
|Record |Bedeutung

|A
|Hostname → IPv4-Adresse

|CNAME
|Alias → anderer Hostname (z.B. www → vmls3.lan.m300.smartlearn.ch)

|MX
|Mail-Server für eine Domain

|NS
|Nameserver der Zone

|SOA
|Start of Authority – Zonen-Metadaten (Serial, TTL, etc.)

|TXT
|Freitext – oft für SPF, DKIM, Domain-Verifizierung
|===

    sudo docker exec dns-webmin-bind-1 bash -c 'cat > /etc/bind/named.conf.local << EOF
    // lokale zonen
    //
    zone "lan.m300.smartlearn.ch" {
        type master;
        file "/etc/bind/db.lan.m300.smartlearn.ch";
    };
    zone "220.168.192.in-addr.arpa" {
        type master;
        file "/etc/bind/db.192.168.220";

    zone "210.168.192.in-addr.arpa" {
        type master;
        file "/etc/bind/db.192.168.210";
    };
