SSH Tunneling

FlagEingangRichtung
-Lmein Rechnerlokal → remote
-RSSH-Serverremote → lokal
# -L: MySQL auf vmLS3 lokal erreichbar
ssh -L 127.0.0.1:13306:127.0.0.1:3306 \
    vmadmin@192.168.220.13

mysql -h 127.0.0.1 -P 13306 -u root -p

# -R: Port 80 auf vmLS3:8123 erreichbar
ssh -R 0.0.0.0:8123:127.0.0.1:80 \
    vmadmin@192.168.220.13

# SSH durch Firewall (LP1→LS3, dann LS3→LP1)
ssh -R 12222:127.0.0.1:22 vmadmin@192.168.220.13
# von vmLS3:
ssh -p 12222 vmadmin@127.0.0.1

# Hintergrund, kein Terminal
ssh -fN -L ... oder -R ...

# Tunnel finden & beenden
ps aux | grep ssh
kill <PID>
GatewayPorts yes in /etc/ssh/sshd_config nötig damit 0.0.0.0 bei -R funktioniert.

DNS – Record-Typen

TypBedeutungBeispiel
AName → IPv4vmls3 IN A 192.168.220.13
CNAMEAliaswww IN CNAME vmls3
MXMailserver@ IN MX 10 mail.ch.
NSNameserver@ IN NS ns.m300.ch.
SOAZonen-InfoPflicht, 1. Eintrag
TXTFreitext@ IN TXT "info"
PTRIP → Name13 IN PTR vmls3.ch.
PTR-Einträge immer mit Punkt am Ende: vmls3.m300.smartlearn.ch.
Bei jeder Zonenänderung: Serial-Nummer erhöhen! Dann systemctl restart named

DNS – BIND9 Installation & Konfiguration

# 1. systemd-resolved deaktivieren (zuerst!)
systemctl stop systemd-resolved
systemctl disable systemd-resolved
rm /etc/resolv.conf
echo "nameserver 8.8.8.8" > /etc/resolv.conf

# 2. Installieren
apt install -y bind9

/etc/bind/named.conf.options

options {
    directory "/var/cache/bind";
    forwarders { 8.8.8.8; };
    allow-recursion { 192.168.210.0/24; 192.168.220.0/24; };
    dnssec-validation no;
    listen-on-v6 { };
    auth-nxdomain no;
    empty-zones-enable no;
};

/etc/bind/named.conf.local

zone "lan.m300.smartlearn.ch" { type master; file "/etc/bind/db.lan.m300.smartlearn.ch"; };
zone "220.168.192.in-addr.arpa" { type master; file "/etc/bind/db.192.168.220"; };
zone "210.168.192.in-addr.arpa" { type master; file "/etc/bind/db.192.168.210"; };

/etc/bind/db.lan.m300.smartlearn.ch

$TTL 3600

Apache2 – Webserver

apt install -y apache2

mkdir -p /var/www/html/myweb
chown -R www-data:www-data /var/www/html/myweb

/etc/apache2/sites-available/myweb.conf

<VirtualHost *:80>
    ServerName   myweb.m300.smartlearn.ch
    DocumentRoot /var/www/html/myweb
    ErrorLog  ${APACHE_LOG_DIR}/myweb-error.log
    CustomLog ${APACHE_LOG_DIR}/myweb-access.log combined
</VirtualHost>
a2ensite myweb.conf
apache2ctl configtest      # Syntax prüfen!
systemctl reload apache2
Name-based Virtual Hosting: Apache schaut auf den HTTP Host-Header. Mehrere .conf-Files = mehrere Sites auf Port 80.
# VHost testen ohne DNS:
curl -H "Host: myweb.m300.smartlearn.ch" http://127.0.0.1
apache2ctl -S              # alle VHosts anzeigen
tail -f /var/log/apache2/error.log

Traefik – Setup

# Verzeichnisse
mkdir -p /m300data/docker-containers/traefik
cd /m300data/docker-containers/traefik
mkdir -p traefik/certs/default data
# cert.pem + privkey.pem nach traefik/certs/default/

# Passwort-Hash für Dashboard
apt install -y apache2-utils
htpasswd -nb admin meinPasswort    # Ausgabe kopieren

traefik/traefik.yml

entryPoints:
  web:
    address: ":80"
  websecure:
    address: ":443"
providers:
  docker:
    exposedByDefault: false
    network: traefik-proxy
  file:
    filename: /etc/traefik/dynamic.yml
    watch: true
api:
  insecure: false

traefik/dynamic.yml (TLS-Zertifikat)

tls:
  stores:
    default:
      defaultCertificate:
        certFile: /etc/traefik/certs/default/cert.pem
        keyFile:  /etc/traefik/certs/default/privkey.pem

docker-compose.yml

services:
  traefik:
    image: "traefik:v3.6"
    container_name: traefik
    restart: unless-stopped
    networks: [traefik-proxy]
    ports: ["80:80", "443:443"]
    volumes:
      - "/var/run/docker.sock:/var/run/docker.sock:ro"
      - "./data:/data"
      - "./traefik:/etc/traefik"
    labels:
      - "traefik.enable=true"
      - "traefik.http.routers.to-https.rule=hostregexp(`.+.m300.smartlearn.ch`)"
      - "traefik.http.routers.to-https.entrypoints=web"
      - "traefik.http.routers.to-https.middlewares=to-https"
      - "traefik.http.middlewares.to-https.redirectscheme.scheme=https"
      - "traefik.http.routers.traefik.rule=Host(`traefik.m300.smartlearn.ch`)"
      - "traefik.http.routers.traefik.entrypoints=websecure"
      - "traefik.http.routers.traefik.tls=true"
      - "traefik.http.routers.traefik.service=api@internal"
      - "traefik.http.routers.traefik.middlewares=auth"
      - "traefik.http.middlewares.auth.basicauth.users=admin:$$apr1$$..."
networks:
  traefik-proxy:
    external: true
docker compose up -d
# Dashboard: https://traefik.m300.smartlearn.ch/dashboard/

Traefik – App anbinden

HTTP (Port 80):

labels:
  - "traefik.enable=true"
  - "traefik.http.routers.APP.rule=Host(`app.m300.smartlearn.ch`)"
  - "traefik.http.routers.APP.entrypoints=web"
  - "traefik.http.services.APP.loadbalancer.server.port=80"
networks:
  - traefik-proxy
  - default

HTTPS (Port 443):

labels:
  - "traefik.enable=true"
  - "traefik.http.routers.APP.rule=Host(`app.m300.smartlearn.ch`)"
  - "traefik.http.routers.APP.entrypoints=websecure"
  - "traefik.http.routers.APP.tls=true"
  - "traefik.http.services.APP.loadbalancer.server.port=80"
ports: aus dem App-compose entfernen wenn Traefik läuft!
App muss im traefik-proxy Netzwerk sein + external: true in networks:
Label-TeilBedeutung
traefik.enable=trueContainer aktivieren
routers.NAME.ruleHostname-Regel
routers.NAME.entrypointsweb oder websecure
routers.NAME.tls=trueHTTPS aktivieren
services.NAME.loadbalancer.server.portContainer-Port intern
